## 🏗️ 架构合规性检查

### 1. 模块放置规范

- [ ] **新模块是否放在了正确的目录？**
  - 彩票相关 → `src/calculations/lottery/`
  - 赔率分析 → `src/calculations/odds/`
  - 量化计算 → `src/calculations/quant/`
  - 回测 → `src/calculations/backtesting/`
  - 结算 → `src/calculations/settlement/`
  - 历史数据 → `src/calculations/history/`
  - 通用工具 → `src/calculations/utils/`
- [ ] **是否有文件直接放在 `src/calculations/` 根目录？（除了 `__init__.py`）**
- [ ] **是否在 `src/` 根目录创建了新的目录？（禁止）**

### 2. 单一真实来源原则

- [ ] **是否硬编码了彩票官方规则？（绝对禁止）**
  - 返奖率（0.69, 0.65）
  - 最大串关数
  - 玩法列表
  - 联赛分类
- [ ] **是否从 `src.calculations.lottery` 统一导入官方规则？**
- [ ] **是否复制粘贴了官方规则代码到其他文件？**

### 3. 重复代码检查

- [ ] **是否有重复的文件名？**
- [ ] **是否有重复的功能实现？**
- [ ] **是否应该使用已有的模块而不是重新实现？**
